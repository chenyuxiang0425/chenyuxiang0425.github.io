# 从选择排序中探索代码实现的一般步骤及单元测试的方法

拿到一个需求，解决需求的一般步骤是为：拆分问题，分段实现，测试先行，实现在后。先从全局架构，把其分为具体细小问题后步步测试，步步完成。

## 问题

用选择排序算法对字符串顺序进行排序，如字符串组`{"i", "have", "an", "egg"}`，排序后为`{"an", "egg", "have", "i"}`。

## 步骤

### 1. 根据需求写测试

程序员需要拥有的最重要的技能之一，就是能够判断代码何时正确。因此拿到需求，首先要做的，就是根据需求写测试用例。通过 Junit 简化测试的编写。

```java
public static void testSort() {
    String[] input = {"i", "have", "an", "egg"};
    String[] expected = {"an", "egg", "have", "i"};
    Sort.sort(input);
    org.junit.Assert.assertArrayEquals(expected, input);
}
```

给`Sort.sort` 一个空方法来测试写的测试用例是否正常运行。

```java
public class Sort {
    /** Sorts strings destructively. */
    public static void sort(String[] x) { 
        
    }
}
```

```java
    at org.junit.internal.ComparisonCriteria.arrayEquals(ComparisonCriteria.java:55)

```

这个运行结果较为难看，但已经满足要求了，之后会把它优化一下。



### 2. 拆分问题

对于拆分后的每一个步骤，都通过新建 `method` 进行解决。

最简单的排序算法是“选择排序”，选择排序包括三个步骤：

- 找到最小的元素。
- 将其移到前面。
- 选择排序剩余的 N-1 个项目。

可以在空白`Sort.sort`方法中写一些简短的注释来指导我们的思考：

```java
public class Sort {
    /** Sorts strings destructively. */
    public static void sort(String[] x) { 
           // find the smallest item
           // move it to the front
           // selection sort the rest (using recursion?)
    }
}
```

对于每一个步骤，都用 helper method 进行解决，首先是第一个步骤，找到最小的元素。

```java
public class Sort {
    /** Sorts strings destructively. */
    public static void sort(String[] x) { 
           // find the smallest item
           // move it to the front
           // selection sort the rest (using recursion?)
    }

    /** Returns the smallest string in x. */
    public static String findSmallest(String[] x) {
        return x[0];
    }
}

```

接下来逐步分别实现这三个步骤。

写第一个 helper method，解决找最小元素的问题。

```java
/** Returns the smallest string in x. 
  * @source Got help with string compares from https://goo.gl/a7yBU5. */
public static String findSmallest(String[] x) {
    String smallest = x[0];
    for (int i = 0; i < x.length; i += 1) {
        int cmp = x[i].compareTo(smallest);
        if (cmp < 0) {
            smallest = x[i];
        }
    }
    return smallest;
}
```

#### 2.2 将其移到前面

写个名为`swap` 的方法实现交换功能。

先编写测试。

```java
public class TestSort {
    ...    

    /** Test the Sort.swap method. */
    public static void testSwap() {
        String[] input = {"i", "have", "an", "egg"};
        int a = 0;
        int b = 2;
        String[] expected = {"an", "have", "i", "egg"};

        Sort.swap(input, a, b);
        org.junit.Assert.assertArrayEquals(expected, input);
    }

    public static void main(String[] args) {
        testSwap();
    }
}
```

这里把第三个元素和第一个元素交换了。

再写名为`swap`的 helper method。

```java
public static void swap(String[] x, int a, int b) {
    String temp = x[a];
    x[a] = x[b];
    x[b] = temp;
}
```

测试中途如果出现问题，可以设置断点进行排查。

#### 2.3 阶段性总结

至此，对于需求：选择排序算法，已经实现了前两个步骤。

1. 获取最小元素
2. 交换两个元素的位置

并且这两个函数通过测试，可以认定是正确的。

观察这两个方法的输入输出参数 ：

```java
public static String findSmallest(String[] x) {
```

`findSmallest`方法接入参数 `String[] x`，输出一个 `String`的返回值。

```java
public static void swap(String[] x, int a, int b) 
```

`swap`方法接入三个参数 `String[] x, int a, int b` ，无返回值输出。

这里看到，我们的 API 不对应，`String`输出无法接入`swap`的参数中。因此要对 `findSmallest`方法进行改写，使其返回值为 `index`。

#### 2.4 改写`findSmallest`方法

改写测试用例

```java
    /** Test the Sort.findSmallest method. */
    public static void testFindSmallest() {
        String[] input = {"i","have","an","egg"};
        int expected = 2;
        int actual = Sort.findSmallest(input);
        org.junit.Assert.assertEquals(expected,actual);

        String[] input2 = {"there","are","some","eggs"};
        int expected2 = 1;
        int actual2 = Sort.findSmallest(input2);

    }

```

改写`findSmallest`方法

```java
    /** Return the Index smallest of the String[] x.  */
     public static int findSmallest(String[] x) {
        int smallestIndex = 0;
        for (int i = 0; i < x.length; i++) {
            int comp = x[i].compareTo(x[smallestIndex]);
            // From the Internet, if x[i] less than x[smallestIndex], comp will be -1.
            if (comp < 0) {
                smallestIndex = i;
            }
        }
        return smallestIndex;
    }
```

通过测试，说明修改后的 helper method 是可用的。

#### 2.5 用 递归辅助方法 选择排序剩余的 N-1 个项目

至此，我们的`sort`方法为以下：

```java
/** Sorts strings destructively. */
public static void sort(String[] x) { 
   int smallestIndex = findSmallest(x);
   swap(x, 0, smallestIndex);
   // recursive call??
}
```

首先想到的是用 Python 中的切片：

```java
/** Sorts strings destructively. */
public static void sort(String[] x) { 
   int smallestIndex = findSmallest(x);
   swap(x, 0, smallestIndex);
   sort(x[1:])
}
```

但 Java 没有对子数组的应用，不能只传递数组中下一项的地址。

Java 中典型的解决方案是创建一个私有 helper method 来实现递归调用，这样可以自定义需要输入的参数，而不是仅仅只有 `String[] x`，创建的名称也为`sort`。

```java
/** Sorts strings destructively starting from item start. */
private static void sort(String[] x, int startIndex) { 
    // TODO
}
```

多给了它一个`startIndex`，来定义起始位置。

用递归写很简单，写递归时记得要先写递归终止条件。

```java
/** Sorts strings destructively starting from item start. */
private static void sort(String[] x, int start) { 
       if (start == x.length) {
       return;
   }
   int smallestIndex = findSmallest(x);
   swap(x, start, smallestIndex);
   sort(x, start + 1);
}
```

有了辅助方法，可以有效进行排序了。

```java
/** Sorts strings destructively. */
public static void sort(String[] x) { 
   sort(x, 0);
}
```

当尝试在非固有递归的数据结构（例如数组）上使用递归时，这种方法非常普遍。

进行测试，发现出错了：

```java
Exception in thread "main" arrays first differed at element [0]; expected:<[an]> but was:<[have]>
	at org.junit.internal.ComparisonCriteria.arrayEquals(ComparisonCriteria.java:55)
	at org.junit.Assert.internalArrayEquals(Assert.java:532)
	at org.junit.Assert.assertArrayEquals(Assert.java:283)
	at org.junit.Assert.assertArrayEquals(Assert.java:298)

```

使用 IntelliJ Debugger进行调试，发现 `findSmallest`方法在对后面的数组进行排序的时候，将其与数组中第一个数字进行了交换。

因此我们要对`findSmallest`进行改写，将其从**获取最小元素**的方法改写为**获取 startIndex 后的最小元素**的方法。

```java
public static int findSmallest(String[] x, int start) {
    int smallestIndex = start;
    for (int i = 0; i < x.length; i += 1) {
        int cmp = x[i].compareTo(x[smallestIndex]);
        if (cmp < 0) {
            smallestIndex = i;
        }
    }
    return smallestIndex;
}
```

**这是一个重大更改，因此需要对所有涉及到该模块的方法重新测试。**

先是 `findSmallest`测试

```java
public static void testFindSmallest() {
    String[] input = {"i", "have", "an", "egg"};
    int expected = 2;

    int actual = Sort.findSmallest(input, 0);
    org.junit.Assert.assertEquals(expected, actual);        

    String[] input2 = {"there", "are", "many", "pigs"};
    int expected2 = 2;

    int actual2 = Sort.findSmallest(input2, 2);
    org.junit.Assert.assertEquals(expected2, actual2);
```

没通过。。翻回去看发现`for`语句中 `i=0` 的初始值忘改了 `i=start`。

```java
public static int findSmallest(String[] x, int start) {
    int smallestIndex = start;
    for (int i = start; i < x.length; i += 1) {
        int cmp = x[i].compareTo(x[smallestIndex]);
        if (cmp < 0) {
            smallestIndex = i;
        }
    }
    return smallestIndex;
}
```

通过。

接下来`Sort.sort`，我们进行修改，以使其在中使用新`start`参数`findSmallest`：

```java
/** Sorts strings destructively starting from item start. */
private static void sort(String[] x, int start) { 
   if (start == x.length) {
       return;
   }
   int smallestIndex = findSmallest(x, start);
   swap(x, start, smallestIndex);
   sort(x, start + 1);
}
```

测试通过。

至此，选择排序方法完成。

## 更好的解决方法

想象中可以直接调用`sort`来验证`findSmallest`是否正确，但实际操作中却往往是灾难性的，因为这会同时在你的大脑中储存太多东西。一组自动化测试能有助于减少这种认知负荷，我们只需要确认`testFindSmallest`成功运行，然后可以尽情地编写`sort`方法。

之前写的测试方法有些笨拙，可以用一些 Junit 的增强功能。

#### 测试注释

第一个增强功能是使用所谓的“测试注释”。为此，我们：

- 在每种方法之前都加`@org.junit.Test`（无分号）。
- 将每种测试方法更改为非静态。
- 从`TestSort`类中删除我们的`main`方法。

这种基于注释的方法具有多个优点：

- 无需手动调用测试。
- 所有测试都在运行，而不仅仅是我们指定的测试。
- 如果一项测试失败，则其他测试仍将运行。
- 提供了运行了多少测试以及通过了多少测试的计数。
- 关于测试失败的错误消息看起来更好看。
- 如果所有测试均通过，我们会收到一条不错的消息，并显示一个绿色条，而不是简单地没有输出。

#### 导入语句

我们首先将import语句添加`import org.junit.Test;`到顶部。这样做了以后，我们可以更换的所有实例`@org.junit.Test`与简单`@Test`。

然后添加第二个import语句`import static org.junit.Assert.*`。完成此操作后，我们可以在任何地方都可以忽略任何地方`org.junit.Assert.`。例如，我们可以替换`org.junit.Assert.assertEquals(expected2, actual2);`为`assertEquals(expected2, actual2);`

## 总结

**绝对**应该**编写测试，但仅当它们可能有用时才写！**



#### 单元测试的优点

1. 使您对代码的每个单元都有信心。

2. 这也有助于减少调试时间，

3. 可以一次将注意力集中在一个代码单元上（通常是一个方法）。

4. 单元测试还迫使您明确每个代码单元应完成的工作。

#### 单元测试的一般步骤Test-Driven Development (TDD)

TDD是一个开发过程，其中我们在编写代码本身之前编写代码测试。步骤如下：

1. 识别新功能。
2. 为此功能编写一个单元测试。
3. 运行测试。它应该失败。
4. 编写通过测试的代码。
5. 可选：重构代码以使其更快，更干净等。我们已经引用了应该通过的测试。

#### 集成测试

集成测试可验证组件之间是否正确交互。

集成测试所面临的挑战是：

1. 手动完成既繁琐又自动化。

2. 但集成测试在较高的抽象水平上，很容易错过细微或罕见的错误。

```java
public class Sort {

    public static void swap(String[] input,int a,int b) {
        String temp;
        temp = input[a];
        input[a] = input[b];
        input[b] = temp;
    }

    /** Return the Index smallest of the String[] x.  */
     public static int findSmallest(String[] x, int start) {
        int smallestIndex = start;
        for (int i = start; i < x.length; i++) {
            int comp = x[i].compareTo(x[smallestIndex]);
            // From the Internet, if x[i] less than x[smallestIndex], comp will be -1.
            if (comp < 0) {

            }
        }
        return smallestIndex;
    }

    private static void sort(String[] x,int startIndex) {
         if (startIndex == x.length) {
             return;

         int smallestIndex = findSmallest(x,startIndex); //
         swap(x,startIndex,smallestIndex);
         sort(x,startIndex + 1);
```
