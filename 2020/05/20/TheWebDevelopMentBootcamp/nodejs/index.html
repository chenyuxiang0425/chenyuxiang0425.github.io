module vs module.exports

  • Node先准备一个对象module,并将其传入加载函数,因此我们在js文件中可以直接调用
  • 真正的函数运行如下： 由上可以看出：
  • 默认情况下，Node准备的exports变量和module.exports变量实际上是同一个变量
    • 并且初始化为空对象{}
  • 因此两种是等价的
    • exports.foo = function () { return 'foo'; };
    • module.exports.foo = function () { return 'foo'; };
  • 但要输出是一个函数或数组，那么，只能给 module.exports赋值：
    • 因为你要输出不同的东西的时候，exports 变了但 module.exports 没变
    • module.exports = function () { return 'foo'; };

结论：

  • 只需要用 module.exports , exports 是一种函数内部的其他变量名的引用
